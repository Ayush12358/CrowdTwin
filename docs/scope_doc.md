# Crowd-Twin: Hackathon Scope 

**Hypothesis:** Move from reactive monitoring (7 p/m²) to proactive control (**15-min lead time**).

### 3 Core Use Cases
- **Verify:** Operator checks predictive alerts and 15-min forecasts.
- **Act:** Staff executes redirection directives (e.g., "Close Gate 4").
- **Plan:** Planner stress-tests layouts to find design pinch-points.

### 10 Functional Screens
1. **Live Heatmap** (Real-time monitoring)
2. **Predictive Mode** (Lead-time forest)
3. **Threshold Alert** (Predicted 7 p/m² warning)
4. **Simulation Playback** (Prediction verification)
5. **Command Panel** (Directive entry)
6. **Staff Terminal** (Mobile instructions)
7. **Outcome Forecast** (Predicted success after action)
8. **Layout Stress-Test** (Arena planning view)
9. **Bottleneck Analysis** (Design-based heatmap)
10. **Safety Analytics** (Executive Incident report)

**Deliverables:** Clickable wireframe + Dummy API stubs.
